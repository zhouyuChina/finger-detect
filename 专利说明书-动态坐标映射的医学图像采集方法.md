# 发明专利说明书

## 一种基于动态坐标映射的医学图像智能采集方法及系统

---

## 【技术领域】

本发明涉及医学影像技术和移动端图像处理领域,具体涉及一种基于动态坐标映射的医学图像智能采集方法及系统,特别适用于微信小程序等移动应用平台中的医学健康检测场景。

---

## 【背景技术】

随着移动互联网和人工智能技术的快速发展,基于智能手机的健康检测应用日益普及。用户通过拍照上传病灶部位照片(如皮肤、指甲、眼睛等),后端AI系统进行智能分析并给出健康建议,已成为远程医疗和个人健康管理的重要手段。

然而,现有技术存在以下问题:

### 1. 坐标系不统一的技术难题

在微信小程序等移动应用平台中,存在三种不同的坐标系统:

**(1) 屏幕UI坐标系**
- 小程序采用rpx(responsive pixel)作为UI布局单位
- 规定屏幕宽度固定为750rpx
- 不同设备的实际分辨率千差万别(从iPhone SE的375×667到iPad Pro的1024×1366像素)
- UI元素(如引导框、遮罩层)的位置和尺寸以rpx为单位定义

**(2) 设备物理像素坐标系**
- 设备屏幕的实际物理分辨率,单位为px(pixel)
- 不同设备的屏幕宽度从320px(iPhone SE)到1366px(iPad横屏)不等
- 存在设备像素比(DPR, Device Pixel Ratio),典型值为1x、2x、3x
- 相机预览画面基于此坐标系进行渲染

**(3) 拍摄图片像素坐标系**
- 微信小程序调用系统相机拍照后,生成的图片分辨率取决于相机硬件
- 典型分辨率范围:1080×1920(普通手机)到4096×3072(高端相机)
- 拍摄图片的宽高比不一定等于屏幕显示区域的宽高比
- 小程序camera组件会自动对拍摄图片进行缩放和居中显示,以适配屏幕

**核心矛盾:**
用户在屏幕上看到的引导框(UI坐标系)与实际拍摄的图片(图片坐标系)之间存在复杂的映射关系。如果直接将UI坐标应用到图片裁剪,会导致裁剪位置严重偏移,无法准确提取目标区域。

### 2. 现有方案的缺陷

**(1) 固定坐标方案**
- 直接使用固定的裁剪坐标(如"从图片中心裁剪300×400像素区域")
- 无法适配不同设备和相机分辨率
- 导致在某些设备上裁剪区域完全错位

**(2) 手动框选方案**
- 拍照后让用户手动拖拽裁剪框选择区域
- 增加用户操作负担,尤其对老年人和医学专业知识缺乏者不友好
- 手动选择精度不高,影响后续AI分析准确性

**(3) 简单比例缩放方案**
- 假设相机预览与拍摄图片的宽高比完全一致
- 未考虑camera组件的自动缩放和居中显示机制
- 在实际应用中裁剪精度差,误差可达20%-50%

**(4) 服务端裁剪方案**
- 上传完整照片到服务器,由服务端AI进行区域检测和裁剪
- 上传大尺寸图片消耗流量和时间(一张4MB照片在4G网络下需5-10秒)
- 增加服务器计算负担
- 用户等待时间长,体验差

### 3. 医学图像采集的特殊要求

医学健康检测对图像质量有严格要求:

- **精确性要求**: 必须准确提取病灶区域(如指甲、皮疹等),避免背景干扰
- **标准化要求**: 不同用户、不同设备采集的图像应保持一致的构图和视角
- **实时性要求**: 采集过程应实时反馈,避免长时间等待
- **易用性要求**: 操作简单直观,无需用户手动调整

现有技术难以同时满足上述要求,亟需一种能够自动、精确、高效地进行医学图像采集的技术方案。

## 【发明内容】

### 一、发明目的

本发明的目的在于解决现有技术中存在的坐标系不统一、裁剪精度低、用户体验差等问题,提供一种基于动态坐标映射的医学图像智能采集方法及系统,实现:

1. 自动建立UI坐标系、设备坐标系、图片坐标系的三层映射关系
2. 精确计算引导框在实际拍摄图片中的对应位置
3. 自动裁剪出目标区域,无需用户手动操作
4. 适配各种设备分辨率和相机配置
5. 提供可视化引导界面,提升用户体验

### 二、技术方案

本发明提供一种基于动态坐标映射的医学图像智能采集方法,包括以下步骤:

#### **步骤S1: 可视化引导界面构建**

在相机预览界面上叠加视觉引导层,包括:

- **镂空遮罩层**: 使用半透明遮罩覆盖整个屏幕,在目标采集区域(如指甲位置)创建透明镂空效果
  - 遮罩透明度: 85%(rgba(0, 0, 0, 0.85))
  - 镂空区域形状: 椭圆形或自定义形状,模拟真实器官形态
  - 实现方式: CSS box-shadow技术,单一DOM元素实现镂空效果

- **引导框标记**: 在镂空区域绘制虚线框或高亮边框,明确标注采集位置
  - 位置参数(相对于屏幕可视区域):
    - 水平位置: 居中对齐(left: 50%, transform: translateX(-50%))
    - 垂直位置: 相对于窗口高度的特定比例(如46%)
    - 宽度: 280rpx(响应式像素单位)
    - 高度: 360rpx
  - 形状参数: 支持椭圆形圆角(border-radius: 140rpx 140rpx 0 0 / 180rpx 180rpx 80rpx 80rpx)

- **文字与图标引导**: 在引导框周围显示操作提示
  - 引导内容包括:"请把指甲放置于框架内"、"确保自然光充足"、"请确保背景没有杂物"
  - 采用图标+文字的组合形式,提升可读性

#### **步骤S2: 多维度系统参数获取**

在用户触发拍照操作后,立即获取以下系统参数:

**(1) 屏幕与窗口信息**
```javascript
const systemInfo = wx.getSystemInfo()
- screenWidth: 屏幕物理宽度(px)
- screenHeight: 屏幕物理高度(px)
- windowHeight: 可视窗口高度(px,不包含系统状态栏、导航栏)
- pixelRatio: 设备像素比(如iPhone为2或3)
```

**(2) 拍摄图片信息**
```javascript
const imageInfo = wx.getImageInfo(tempImagePath)
- imageWidth: 图片实际宽度(px)
- imageHeight: 图片实际高度(px)
- orientation: 图片方向(正常/旋转90°/180°等)
- type: 图片格式(JPEG/PNG)
```

**(3) UI布局参数**
- rpx到px转换因子: `rpxToPixel = screenWidth / 750`
- 引导框在屏幕上的物理尺寸:
  - `guideFrameWidth = 280 × rpxToPixel`(px)
  - `guideFrameHeight = 360 × rpxToPixel`(px)
- 引导框在屏幕上的物理位置:
  - `guideFrameX = (screenWidth - guideFrameWidth) / 2`(水平居中)
  - `guideFrameY = windowHeight × 0.46 - guideFrameHeight / 2`(垂直46%位置)

#### **步骤S3: 动态坐标映射算法**

建立从屏幕坐标到图片坐标的映射关系,核心在于处理宽高比不一致的情况:

**(1) 计算宽高比**
```javascript
const screenRatio = screenWidth / windowHeight  // 屏幕宽高比
const imageRatio = imageWidth / imageHeight     // 图片宽高比
```

**(2) 确定缩放模式**

微信小程序camera组件的显示规则:保持图片原始宽高比,等比缩放至完全覆盖屏幕可视区域,多余部分裁剪隐藏。

**情况A: 图片更宽(imageRatio > screenRatio)**
- 以高度为基准进行缩放
- 图片的高度缩放至与窗口高度相等
- 图片的宽度会超出屏幕宽度,左右两侧被裁剪

计算公式:
```javascript
scaleY = imageHeight / windowHeight  // 垂直缩放因子
scaleX = scaleY                       // 水平缩放因子(保持宽高比)
offsetX = (imageWidth - screenWidth × scaleX) / 2  // 水平方向的偏移量
offsetY = 0                           // 垂直方向无偏移
```

**情况B: 图片更高(imageRatio ≤ screenRatio)**
- 以宽度为基准进行缩放
- 图片的宽度缩放至与屏幕宽度相等
- 图片的高度会超出窗口高度,上下两侧被裁剪

计算公式:
```javascript
scaleX = imageWidth / screenWidth     // 水平缩放因子
scaleY = scaleX                        // 垂直缩放因子(保持宽高比)
offsetX = 0                            // 水平方向无偏移
offsetY = (imageHeight - windowHeight × scaleY) / 2  // 垂直方向的偏移量
```

**(3) 坐标映射计算**

将引导框的屏幕坐标映射到图片坐标:
```javascript
actualCropX = guideFrameX × scaleX + offsetX
actualCropY = guideFrameY × scaleY + offsetY
actualCropWidth = guideFrameWidth × scaleX
actualCropHeight = guideFrameHeight × scaleY
```

**(4) 边界保护**

防止裁剪区域超出图片边界:
```javascript
actualCropX = Math.max(0, actualCropX)
actualCropY = Math.max(0, actualCropY)
actualCropWidth = Math.min(imageWidth - actualCropX, actualCropWidth)
actualCropHeight = Math.min(imageHeight - actualCropY, actualCropHeight)
```

#### **步骤S4: 精确图像裁剪**

使用Canvas API进行高精度裁剪:

**(1) 创建离屏Canvas**
- Canvas尺寸设置为裁剪区域的实际尺寸
- 使用隐藏Canvas(位置设置为left: -9999px),避免影响UI显示

**(2) 绘制裁剪区域**
```javascript
ctx.drawImage(
  imagePath,                              // 源图片路径
  actualCropX,                            // 源图片裁剪起始X坐标
  actualCropY,                            // 源图片裁剪起始Y坐标
  actualCropWidth,                        // 源图片裁剪宽度
  actualCropHeight,                       // 源图片裁剪高度
  0,                                      // Canvas绘制起始X坐标
  0,                                      // Canvas绘制起始Y坐标
  actualCropWidth,                        // Canvas绘制宽度
  actualCropHeight                        // Canvas绘制高度
)
```

**(3) 导出裁剪结果**
```javascript
wx.canvasToTempFilePath({
  canvasId: 'cropCanvas',
  x: 0, y: 0,
  width: actualCropWidth,
  height: actualCropHeight,
  destWidth: actualCropWidth,    // 输出图片宽度
  destHeight: actualCropHeight,  // 输出图片高度
  fileType: 'jpg',               // 输出格式
  quality: 0.9                   // 图片质量(0-1)
})
```

#### **步骤S5: 质量验证与优化**

对裁剪后的图片进行质量检查:

**(1) 尺寸验证**
- 检查裁剪后的图片尺寸是否符合预期
- 若裁剪区域过小(如宽度或高度小于100px),提示用户重新拍摄

**(2) 图片压缩**
- 若裁剪后的图片尺寸过大(如超过2MB),进行智能压缩
- 压缩策略:保持宽高比,限制最大边长为1080px,质量为80%

**(3) 异常处理**
- 裁剪失败时,使用原始图片作为备用方案
- 向用户展示提示信息:"图片处理失败,使用原图"

#### **步骤S6: 结果返回与后续处理**

将裁剪后的图片传递给后续流程:

- 本地预览:在预览页面显示裁剪后的图片,供用户确认
- 数据传递:通过页面路由参数或全局状态管理,将图片路径传递给检测页面
- AI分析:上传裁剪后的图片到服务器进行AI检测

### 三、技术创新点

### 实施例1: 指甲健康检测应用

#### 1.1 应用场景

开发一款微信小程序,用于检测灰指甲等指甲疾病。用户打开相机,将指甲对准屏幕上的引导框,拍照后自动裁剪出指甲区域,上传到服务器进行AI分析。

#### 1.2 界面设计

#### 1.3 核心算法实现

**(1) 拍照触发函数**

```javascript
// 用户点击拍照按钮
takePhoto() {
  const ctx = wx.createCameraContext()
  ctx.takePhoto({
    quality: 'high',  // 高质量拍摄
    success: (res) => {
      // 拍照成功,立即进行裁剪
      this.cropPhoto(res.tempImagePath)
    },
    fail: (err) => {
      wx.showToast({ title: '拍照失败', icon: 'none' })
    }
  })
}
```

**(2) 裁剪核心算法**

```javascript
async cropPhoto(originalImagePath) {
  try {
    wx.showLoading({ title: '处理中...' })

    // 【步骤1】获取系统信息
    const systemInfo = await wx.getSystemInfo()
    const screenWidth = systemInfo.screenWidth      // 如:375px (iPhone X)
    const screenHeight = systemInfo.screenHeight    // 如:812px
    const windowHeight = systemInfo.windowHeight    // 如:724px (扣除状态栏)

    // 【步骤2】获取图片信息
    const imageInfo = await wx.getImageInfo({ src: originalImagePath })
    const imageWidth = imageInfo.width     // 如:3024px (12MP相机)
    const imageHeight = imageInfo.height   // 如:4032px

    // 【步骤3】计算引导框在屏幕上的位置
    const rpxToPixel = screenWidth / 750   // rpx转px因子,如 375/750=0.5
    const guideFrameWidth = 280 * rpxToPixel    // 280rpx → 140px
    const guideFrameHeight = 360 * rpxToPixel   // 360rpx → 180px
    const guideFrameX = (screenWidth - guideFrameWidth) / 2  // 水平居中,如 (375-140)/2=117.5px
    const guideFrameY = windowHeight * 0.46 - guideFrameHeight / 2  // 垂直46%位置,如 724*0.46-90=243px

    // 【步骤4】计算缩放关系
    const screenRatio = screenWidth / windowHeight  // 如 375/724=0.518
    const imageRatio = imageWidth / imageHeight     // 如 3024/4032=0.75

    let scaleX, scaleY, offsetX = 0, offsetY = 0

    if (imageRatio > screenRatio) {
      // 图片更宽(0.75 > 0.518),以高度为基准
      scaleY = imageHeight / windowHeight  // 如 4032/724=5.57
      scaleX = scaleY
      offsetX = (imageWidth - screenWidth * scaleX) / 2  // 如 (3024-375*5.57)/2=466px
    } else {
      // 图片更高,以宽度为基准
      scaleX = imageWidth / screenWidth    // 如 3024/375=8.064
      scaleY = scaleX
      offsetY = (imageHeight - windowHeight * scaleY) / 2  // 如 (4032-724*8.064)/2=106px
    }

    // 【步骤5】映射到图片坐标
    const actualCropX = Math.max(0, guideFrameX * scaleX + offsetX)
    // 如 117.5 * 5.57 + 466 = 1120px (在原图中的X坐标)

    const actualCropY = Math.max(0, guideFrameY * scaleY + offsetY)
    // 如 243 * 5.57 + 0 = 1353px (在原图中的Y坐标)

    const actualCropWidth = Math.min(imageWidth - actualCropX, guideFrameWidth * scaleX)
    // 如 min(3024-1120, 140*5.57) = min(1904, 780) = 780px

    const actualCropHeight = Math.min(imageHeight - actualCropY, guideFrameHeight * scaleY)
    // 如 min(4032-1353, 180*5.57) = min(2679, 1003) = 1003px

    // 【步骤6】执行裁剪
    const croppedImagePath = await this.performCrop(
      originalImagePath,
      actualCropX,      // 1120
      actualCropY,      // 1353
      actualCropWidth,  // 780
      actualCropHeight  // 1003
    )

    wx.hideLoading()

    // 【步骤7】保存结果
    this.setData({
      photoPath: croppedImagePath,
      showCamera: false
    })

  } catch (error) {
    // 异常处理:使用原图
    this.setData({
      photoPath: originalImagePath,
      showCamera: false
    })
    wx.showToast({ title: '图片处理失败,使用原图', icon: 'none' })
  }
}
```

**(3) Canvas裁剪实现**

```javascript
performCrop(imagePath, x, y, width, height) {
  return new Promise((resolve, reject) => {
    const ctx = wx.createCanvasContext('cropCanvas', this)

    // 设置Canvas尺寸为裁剪区域大小
    this.setData({
      canvasWidth: Math.round(width),   // 如 780px
      canvasHeight: Math.round(height)  // 如 1003px
    })

    // 等待Canvas尺寸更新
    setTimeout(() => {
      // 绘制裁剪区域到Canvas
      ctx.drawImage(
        imagePath,                    // 源图片路径
        Math.round(x),                // 源图片起始X: 1120
        Math.round(y),                // 源图片起始Y: 1353
        Math.round(width),            // 源图片裁剪宽度: 780
        Math.round(height),           // 源图片裁剪高度: 1003
        0,                            // Canvas绘制起始X: 0
        0,                            // Canvas绘制起始Y: 0
        Math.round(width),            // Canvas绘制宽度: 780
        Math.round(height)            // Canvas绘制高度: 1003
      )

      ctx.draw(false, () => {
        // 等待绘制完成
        setTimeout(() => {
          // 导出Canvas为图片
          wx.canvasToTempFilePath({
            canvasId: 'cropCanvas',
            x: 0, y: 0,
            width: Math.round(width),
            height: Math.round(height),
            destWidth: Math.round(width),
            destHeight: Math.round(height),
            fileType: 'jpg',
            quality: 0.9,
            success: (res) => {
              resolve(res.tempFilePath)  // 返回裁剪后的临时文件路径
            },
            fail: reject
          }, this)
        }, 300)
      })
    }, 100)
  })
}
```
