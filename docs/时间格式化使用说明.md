# 时间格式化使用说明

本项目已统一所有时间显示方式，使用 `utils/common.js` 中的时间格式化方法。

## 可用的时间格式化方法

### 1. formatTime(date, format)
**基础时间格式化方法**，支持自定义格式。

```javascript
const common = require('../../utils/common.js')

// 默认格式：YYYY-MM-DD HH:mm:ss
common.formatTime(new Date()) // "2025-01-15 14:30:25"

// 自定义格式
common.formatTime(new Date(), 'YYYY-MM-DD') // "2025-01-15"
common.formatTime(new Date(), 'HH:mm') // "14:30"
common.formatTime(new Date(), 'MM-DD HH:mm') // "01-15 14:30"
```

**支持的格式占位符：**
- `YYYY` - 4位年份
- `MM` - 2位月份
- `DD` - 2位日期
- `HH` - 2位小时
- `mm` - 2位分钟
- `ss` - 2位秒数

---

### 2. formatTimeRelative(timeStr)
**智能相对时间**，最友好的中文显示方式。

**显示规则：**
- 1分钟内：`刚刚`
- 1小时内：`X分钟前`
- 6小时内：`X小时前`
- 当天6小时后：`今天 14:30`
- 昨天：`昨天 14:30`
- 一周内：`周一 14:30`
- 今年其他时间：`01-15 14:30`
- 往年：`2024-01-15`

```javascript
const common = require('../../utils/common.js')

common.formatTimeRelative('2025-01-15T14:28:00') // "2分钟前"
common.formatTimeRelative('2025-01-15T10:00:00') // "今天 10:00"
common.formatTimeRelative('2025-01-14T14:30:00') // "昨天 14:30"
common.formatTimeRelative('2025-01-13T14:30:00') // "周一 14:30"
common.formatTimeRelative('2025-01-01T14:30:00') // "01-01 14:30"
common.formatTimeRelative('2024-12-01T14:30:00') // "2024-12-01"
```

**适用场景：**
- 社交消息时间
- 档案更新时间
- 动态发布时间

---

### 3. formatTimeShort(timeStr)
**简短相对时间**，适合列表展示。

**显示规则：**
- 1分钟内：`刚刚`
- 1小时内：`X分钟前`
- 24小时内：`X小时前`
- 30天内：`X天前`
- 30天后：`YYYY-MM-DD`

```javascript
const common = require('../../utils/common.js')

common.formatTimeShort('2025-01-15T14:28:00') // "2分钟前"
common.formatTimeShort('2025-01-15T12:00:00') // "2小时前"
common.formatTimeShort('2025-01-10T14:30:00') // "5天前"
common.formatTimeShort('2024-12-01T14:30:00') // "2024-12-01"
```

**适用场景：**
- 评论时间
- 反馈时间
- 系统消息时间

---

### 4. formatTimeChinese(timeStr, includeTime)
**完整中文格式**，最正式的显示方式。

```javascript
const common = require('../../utils/common.js')

// 包含时间（默认）
common.formatTimeChinese('2025-01-15T14:30:25')
// "2025年1月15日 14:30:25"

// 仅日期
common.formatTimeChinese('2025-01-15T14:30:25', false)
// "2025年1月15日"
```

**适用场景：**
- 报告生成时间
- 证书有效期
- 正式文档

---

### 5. formatTimeLocale(timeStr, options)
**本地化格式**，使用浏览器本地化设置。

```javascript
const common = require('../../utils/common.js')

// 默认：完整日期时间
common.formatTimeLocale('2025-01-15T14:30:25')
// "2025/01/15 14:30"

// 仅日期
common.formatTimeLocale('2025-01-15T14:30:25', {
  year: 'numeric',
  month: '2-digit',
  day: '2-digit'
})
// "2025/01/15"

// 仅时间
common.formatTimeLocale('2025-01-15T14:30:25', {
  hour: '2-digit',
  minute: '2-digit',
  second: '2-digit'
})
// "14:30:25"
```

**适用场景：**
- 需要严格本地化的场景
- 检测记录时间
- 档案详情时间

## 各页面使用的格式化方法

| 页面 | 使用方法 | 说明 |
|-----|---------|------|
| index.js | `formatTime('MM-DD HH:mm')` | 资讯发布时间 |
| records-compare.js | `formatTimeRelative()` | 档案更新时间 |
| record-gallery.js | `formatTimeLocale()` | 检测记录时间 |
| my-profile.js | `formatTimeLocale()` | 用户档案更新时间 |
| message.js | `formatTime('YYYY-MM-DD')` | 消息列表日期 |
| message-detail.js | `formatTime('YYYY-MM-DD HH:mm')` | 消息详情时间 |

## 迁移指南

如果你在新页面中需要显示时间，请按照以下步骤：

### 1. 引入 common.js
```javascript
const common = require('../../utils/common.js')
```

### 2. 选择合适的方法
根据你的场景选择上述5种方法之一。

### 3. 替换旧代码
```javascript
// ❌ 旧代码
formatTime(timeStr) {
  const date = new Date(timeStr)
  return date.toLocaleString('zh-CN')
}

// ✅ 新代码
// 直接使用 common.formatTimeRelative()
updateTime: common.formatTimeRelative(archive.updatedAt)
```

---

## 注意事项

1. **不要在页面中自定义 formatTime 方法**
   所有页面应该使用统一的格式化方法。

2. **选择合适的方法**
   根据显示场景选择最合适的格式化方法，保持用户体验一致。

3. **时间参数格式**
   所有方法都支持以下时间格式：
   - ISO 8601 字符串：`"2025-01-15T14:30:25"`
   - 时间戳：`1736931025000`
   - Date 对象：`new Date()`

4. **错误处理**
   所有方法都包含了错误处理，如果传入无效时间会返回原始字符串或空字符串。

## 示例代码

### 在列表中显示档案更新时间
```javascript
formatArchives(archives) {
  return archives.map(archive => ({
    id: archive.id,
    name: archive.name,
    updateTime: common.formatTimeRelative(archive.updatedAt)
  }))
}
```

### 在详情页显示检测时间
```javascript
formatDetection(detection) {
  return {
    id: detection.id,
    detectionTime: common.formatTimeLocale(detection.createdAt)
  }
}
```

### 在消息中显示发布时间
```javascript
formatMessage(message) {
  return {
    id: message.id,
    publishTime: common.formatTime(new Date(message.publishedAt), 'MM-DD HH:mm')
  }
}
```
