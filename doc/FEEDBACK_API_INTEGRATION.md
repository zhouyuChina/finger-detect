## API接口对应关系

### 1. 获取系统回复列表
- **接口**: `GET /api/miniprogram/feedback`
- **参数**: `page`, `pageSize`, `status`, `type`
- **响应**: 包含 `list` 数组和 `pagination` 对象

### 2. 提交留言
- **接口**: `POST /api/miniprogram/feedback`
- **参数**: `type: 'bug'`, `title`, `content`
- **响应**: 包含反馈ID和状态信息

**支持的类型**：
- `bug` - 问题
- `suggestion` - 建议  
- `complaint` - 投诉

**当前设置**：所有留言统一使用 `type: 'bug'`

### 3. 获取系统回复详情
- **接口**: `GET /api/miniprogram/feedback/{id}`
- **参数**: 通过查询参数传递ID
- **响应**: 包含完整的回复信息

## 使用说明

### 1. 查看系统回复列表
1. 进入反馈页面
2. 切换到"系统回复"标签页
3. 列表会自动加载系统回复给当前用户的消息
4. 支持下拉刷新和上拉加载更多

### 2. 给系统发送留言
1. 在反馈页面切换到"给系统留言"标签页
2. 填写留言标题（必填，最多100字符）
3. 填写留言内容（必填，最多1000字符）
4. 点击"提交留言"按钮

### 3. 查看系统回复详情
1. 在系统回复列表中点击任意回复项
2. 弹窗显示完整的回复信息
3. 包括标题、内容、图片、状态、官方回复等

## 注意事项

1. **图片上传**: 使用检测模块的上传接口，确保图片格式和大小符合要求
2. **字符限制**: 标题最多100字符，内容最多1000字符
3. **状态显示**: 根据API返回的status字段显示对应状态
4. **错误处理**: 所有网络请求都有完整的错误处理和用户提示
5. **数据刷新**: 提交成功后会自动刷新反馈列表
