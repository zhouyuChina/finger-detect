# 微信开发者工具已知问题说明

## 当前遇到的问题

### 1. AB测试配置API警告
```
[system] clicfg_appbrand_android_enable_aiad_debug_button failed 
{errMsg: "getABTestConfig:fail 开发者工具暂时不支持此 API 调试，请使用真机进行开发"}
```

**问题原因：**
- 这是微信开发者工具的一个已知问题
- AB测试相关的API在开发者工具中不被支持
- 这是微信官方的限制，不是我们代码的问题

**解决方案：**
1. **忽略此警告** - 这个警告不会影响应用功能
2. **使用真机调试** - 如果需要测试AB测试功能，请使用真机
3. **关闭相关功能** - 在开发阶段可以暂时关闭AB测试功能

**影响范围：**
- ✅ 不影响应用核心功能
- ✅ 不影响API调用
- ✅ 不影响页面渲染
- ✅ 不影响用户交互

### 2. 其他常见开发者工具问题

#### 网络请求问题
```
- 本地开发时，某些网络请求可能被拦截
- 建议在 project.private.config.json 中设置 "urlCheck": false
```

#### 组件加载问题
```
- 第三方组件可能加载缓慢
- 建议使用 npm 构建优化
```

#### 缓存问题
```
- 开发者工具缓存可能导致页面不更新
- 建议定期清理缓存或重启开发者工具
```

## 开发建议

### 1. 开发环境配置
```json
// project.private.config.json
{
  "setting": {
    "compileHotReLoad": true,
    "urlCheck": false
  },
  "libVersion": "development"
}
```

### 2. 调试技巧
1. **使用真机调试** - 某些功能只能在真机上测试
2. **定期清理缓存** - 避免缓存导致的问题
3. **查看控制台日志** - 关注重要的错误信息
4. **使用断点调试** - 复杂逻辑建议使用断点

### 3. 性能优化
1. **启用代码压缩** - 生产环境必须启用
2. **优化图片资源** - 压缩图片大小
3. **减少不必要的请求** - 合理使用缓存
4. **组件按需加载** - 使用 lazyCodeLoading

## 生产环境注意事项

### 1. 域名配置
- 确保所有API域名已在微信后台配置
- 生产环境必须使用HTTPS协议

### 2. 版本管理
- 合理设置版本号和构建号
- 及时更新基础库版本

### 3. 错误监控
- 配置错误上报机制
- 监控关键指标

## 常见问题排查

### 1. 页面无法加载
- 检查页面路径是否正确
- 检查页面文件是否存在
- 检查语法错误

### 2. API请求失败
- 检查网络连接
- 检查域名配置
- 检查请求参数

### 3. 组件显示异常
- 检查组件是否正确引入
- 检查组件版本兼容性
- 检查样式冲突
